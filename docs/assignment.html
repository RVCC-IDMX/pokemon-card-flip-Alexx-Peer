Pokemon Card Flip:
DOM Manipulation Assignment

Overview

In this assignment, you'll build an interactive Pokemon Card Flip application using JavaScript DOM manipulation. The application connects to the PokeAPI to fetch Pokemon data, creates interactive card elements, and provides a smooth flipping animation when users interact with the cards. Throughout this project, you'll learn and practice key web development concepts like DOM manipulation, event handling, asynchronous JavaScript, and working with external APIs.

Learning Objectives

  • Create and manipulate DOM elements dynamically
  • Implement event delegation for efficient event handling
  • Use CSS transitions for smooth animations
  • Work with the fetch API and handle asynchronous operations
  • Process and display data from an external API
  • Implement loading states for better user experience
  • Store and retrieve data using element datasets

Project Structure

The project is organized into several modules, each focusing on specific functionality:

Module File Description
Application Logic app.js Main application logic for creating and managing Pokemon cards
Pokemon Service pokemon.js Functions for fetching and processing Pokemon data from the PokeAPI
HTML Structure index.html Base HTML structure with containers for cards and loading spinner
Styling styles.css CSS styles for cards, animations, and layout
Assets assets/ Images such as Pokeball for card fronts

Getting Started

  1. Clone the repository
  2. Open the project in your code editor
  3. Start by examining the provided HTML and CSS files
  4. Complete the code challenges in app.js and pokemon.js marked with // CHALLENGE X: comments
  5. Test your implementation by opening index.html in a browser
Tip: Complete the challenges in order as they build upon each other. The Pokemon Service module (pokemon.js) should be completed first, followed by the main Application Logic (app.js).

Challenge List

Pokemon Service Module (pokemon.js)

Challenge 1: Implement fetchRandomPokemon function

Create a function that fetches a random Pokemon from the PokeAPI. The function should:

  • Generate a random Pokemon ID between 1 and TOTAL_POKEMON
  • Use the fetch API to request data from the PokeAPI
  • Check if the response is successful
  • Process the raw data using the processPokemonData function
  • Return the processed Pokemon data or null if there's an error
  • Handle errors appropriately with try/catch

Challenge 2: Implement fetchMultipleRandomPokemon function

Create a function that fetches multiple random Pokemon at once. The function should:

  • Accept a count parameter for the number of Pokemon to fetch
  • Create an array of promises for fetching random Pokemon
  • Use Promise.all to wait for all promises to resolve
  • Return the array of Pokemon data
  • Handle errors appropriately with try/catch

Challenge 3: Implement processPokemonData function

Create a function that processes the raw Pokemon data into a more usable format. The function should:

  • Extract relevant information from the raw API data
  • Transform data into appropriate formats (e.g., capitalizing names, converting units)
  • Map complex structures to simpler ones (e.g., extracting type names from the types array)
  • Return a well-structured Pokemon object

Challenge 4: Implement findStat function

Create a helper function to find a specific stat from the Pokemon stats array. The function should:

  • Use the Array.find method to locate the stat with the matching name
  • Return the base_stat value if found, or 0 if not found

Challenge 5: Implement capitalizeFirstLetter function

Create a helper function to capitalize the first letter of a string and replace hyphens with spaces. The function should:

  • Take a string as input
  • Capitalize the first character
  • Replace any hyphens with spaces
  • Return the formatted string

Challenge 6: Implement PokemonService export

Create a global PokemonService object to expose the necessary functions. The export should:

  • Attach the PokemonService object to the window object
  • Include the fetchRandomPokemon and fetchMultipleRandomPokemon functions
  • Make the service accessible to other scripts

Application Logic Module (app.js)

Challenge 7: Implement initApp function

Create the main initialization function for the application. The function should:

  • Show the loading spinner
  • Hide the card grid initially
  • Call createCardElements to create the card containers
  • Call fetchAndAssignPokemon to load Pokemon data
  • Set up event listeners
  • Hide the loading spinner and show the card grid when ready
  • Handle the asynchronous flow properly with await

Challenge 8: Implement createCardElements function

Create a function that populates the grid with card elements. The function should:

  • Clear any existing cards in the grid
  • Reset the cards array
  • Create new card elements for the specified card count
  • Append each card to the card grid
  • Store references to the cards in the cards array

Challenge 9: Implement createCardElement function

Create a function that generates a single card element with the proper structure. The function should:

  • Create the main card container element
  • Create the card inner, front, and back elements
  • Add the Pokeball image to the card front
  • Assemble the elements in the correct hierarchy
  • Set appropriate classes and data attributes
  • Return the fully constructed card element

Challenge 10: Implement fetchAndAssignPokemon function

Create a function that fetches Pokemon data and assigns it to cards. The function should:

  • Use the PokemonService to fetch random Pokemon
  • Simulate a loading delay if the debug flag is enabled
  • Assign each Pokemon to the corresponding card
  • Handle errors appropriately with try/catch

Challenge 11: Implement assignPokemonToCard function

Create a function that updates a card with Pokemon data. The function should:

  • Store the Pokemon data in the card's dataset
  • Clear existing content from the card back
  • Create and add elements for the Pokemon image, name, and types
  • Create and add elements for the Pokemon stats (height, weight, abilities)
  • Assemble all elements in the appropriate structure
  • Handle null/undefined inputs gracefully

Challenge 12: Implement handleCardClick function

Create a function that handles click events on cards. The function should:

  • Find the parent card element if a child element was clicked
  • Toggle the 'flipped' class on the card
  • Exit early if no valid card was found

Challenge 13: Implement setupEventListeners function

Create a function that sets up all event listeners. The function should:

  • Add a click event listener to the card grid
  • Use event delegation to handle clicks on child elements
  • Use the handleCardClick function as the event handler

Challenge 14: Implement showLoading and hideLoading functions

Create functions to manage the loading spinner's visibility. The functions should:

  • Show the loading spinner by removing the 'hidden' class
  • Hide the loading spinner by adding the 'hidden' class

Challenge 15: Initialize the application

Add the code to initialize the application when the DOM is loaded. This should:

  • Add an event listener for the 'DOMContentLoaded' event
  • Call the initApp function when the DOM is ready

Testing Your Implementation

As you complete each challenge, test your implementation by opening the index.html file in a browser. You should see:

  • A loading spinner initially
  • A grid of cards displaying Pokeballs when loading completes
  • Cards that flip when clicked, revealing Pokemon details
  • Proper display of Pokemon names, types, and stats
  • Smooth animation when flipping cards
Tip: Use your browser's developer tools (F12 or Right-click → Inspect) to:
  • Check the console for any errors
  • Inspect the DOM structure
  • View network requests to the PokeAPI
  • Test your application on different screen sizes

Extension Challenges

If you complete all the main challenges and want to extend your project, here are some additional features to implement:

Extension 1: Add Filter by Type

Create a type filter that allows users to filter Pokemon cards by type (e.g., Fire, Water, Electric).

  • Add buttons or a dropdown for each Pokemon type
  • Update the card display to show only matching Pokemon
  • Add visual feedback for the active filter

Extension 2: Implement "Catch Pokemon" Functionality

Add a feature to let users "catch" Pokemon and save them to a collection.

  • Add a "Catch" button to each Pokemon card
  • Store caught Pokemon in localStorage
  • Create a section to view caught Pokemon
  • Add the ability to release Pokemon from the collection

Extension 3: Add Pokemon Details Modal

Create a modal dialog that shows more detailed information when a Pokemon is clicked.

  • Design and implement a modal component
  • Display additional Pokemon stats and information
  • Fetch and display Pokemon evolution chain data
  • Add animations for opening/closing the modal

Extension 4: Implement Card Search

Add a search functionality to find specific Pokemon by name.

  • Create a search input field
  • Filter displayed cards as the user types
  • Add visual feedback for no results
  • Implement debouncing to avoid excessive filtering

Extension 5: Add Sound Effects

Enhance the user experience by adding sound effects.

  • Play a flip sound when a card is turned
  • Play Pokemon cries when a card is revealed
  • Add background music with volume controls
  • Ensure sounds can be muted/disabled
